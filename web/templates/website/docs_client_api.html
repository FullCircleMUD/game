Client API

Full Circle MUD communicates over WebSocket using a simple JSON protocol. Any client that can open a WebSocket and send/receive JSON can connect to the game.

The built-in browser client uses this same protocol. This page documents everything you need to build your own client — whether that's a custom UI, a mobile app, a Discord bot, or an AI agent.

{# ── Connecting ── #}

Connecting

Open a WebSocket connection to:

ws://<host>:<port>/websocket

The default port is 4002 for WebSocket traffic. In production this is typically served behind a reverse proxy on wss:// (TLS).

Optional Query Parameters

The browser client appends three query parameters to the URL for session management: 

ws://host:4002/websocket?<csessid>&<cuid>&<browser>
ParameterPurposeRequired?
csessidDjango browser session ID (enables auto-login on reconnect)No
cuidUnique page/tab identifierNo
browserBrowser identifier string (e.g. "chrome", "firefox")No

Custom clients can omit all three — the server handles connections without them. They are only needed if you want browser session auto-login.

{# ── Message Format ── #}

Message Format

All messages — in both directions — are JSON arrays with exactly three elements: 

["cmdname", [args], {kwargs}]
ElementTypeDescription
cmdnamestringThe command or message type
argsarrayPositional arguments
kwargsobjectKeyword arguments

This format is used for everything: game commands, server output, prompts, and out-of-band (OOB) messages.

{# ── Client → Server ── #}

Client → Server Messages

Sending a Game Command

To send a command the player has typed (e.g. "look", "attack goblin", "say hello"):

["text", ["look"], {}]

The first element of args is the raw command string. The server interprets it exactly as if the player typed it at a prompt.

Clean Disconnect

To close the connection gracefully:

["websocket_close", [], {}]
{# ── Server → Client ── #}

Server → Client Messages

Text Output

Room descriptions, combat output, chat messages, and all other game text arrives as: 

["text", ["<HTML-formatted text>"], {}]

The text in args[0] is HTML-formatted. ANSI colour codes from the game engine are converted to HTML <span> tags with CSS classes. Clients that render HTML can display colours directly. Clients that want plain text should strip HTML tags and decode entities.

Prompt

Status information (health, mana, location, etc.) is sent as a prompt: 

["prompt", ["HP: 100/100 | Mana: 50/50"], {}]

Prompts are sent separately from text so clients can display them in a dedicated status area rather than inline with scrolling output.

{# ── Authentication ── #}

Authentication

Authentication uses cryptographic wallet signing:

  1. Client sends ["text", ["connect"], {}]
  2. Server responds with a link that the player can open in a new tab
  3. The link displays a QR code for the player's Xaman wallet
  4. Player scans the QR code and signs in Xaman, proving ownership of the wallet
  5. Server verifies the signature and logs in the matching account

No passwords are used. Your wallet address is your identity — the signed payload proves ownership without ever exposing a private key.

{# ── HTML Processing ── #}

HTML Processing

The server converts ANSI colour codes to HTML before sending over WebSocket. A message like: 

|rYou have been hit!|n

arrives as:

<span class="color-001">You have been hit!</span>
Colour Classes

CSS classes follow the pattern color-NNN for foreground and bgcolor-NNN for background, where NNN is the xterm-256 colour index. Basic ANSI colours map to indices 0–15.

Stripping HTML

If your client doesn't render HTML, strip tags and decode entities. In Python: 

from html.parser import HTMLParser
from html import unescape

class Stripper(HTMLParser):
    def __init__(self):
        super().__init__()
        self.parts = []
    def handle_data(self, data):
        self.parts.append(data)
    def get_text(self):
        return unescape("".join(self.parts))

s = Stripper()
s.feed(html_text)
plain_text = s.get_text()
